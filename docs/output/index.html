FirstTap API documentation version 1

http://firsttap.appspot.com/v1

Welcome

Welcome to the FirstTap API, your seamless payment system for employees at Bows F1.

NB this system does not take payment itself, it simply stores a user's balance and payments against their record. The API must always be accessed using the provided login credentials in auth headers to ensure that only the designated kiosks have access.

Individual user logins also generate session tokens which will expire after 5 minutes.

For information on how we store data, please visit: Privacy Policy To contact a member of the team, please visit: Contact Us

/signup

Sign up new users using existing employee id

/signup post

post

Store a new employee in the FirstTap system

This assumes a unique id for each user and stores them in the system DB. Core employee data is required and the route will fail if values are missing from the body.

The employee name and a usage token valid for 5 minutes will be returned. This must be included in the headers of all further API calls.

/topup

Top up the employee's balance by a given amount

/topup post

post

Top up an employee's balance

Given a valid employeeId and session token, add the given amount to the user's balance.

Known limitations

  • This is not configured to use any specific currency
  • The datastore allows 2 decimal places and figures up to 8 digits (so max £999999.99).

Should this prove insufficient, please contact us for an extension.

post /topup

Top up an employee's balance

Given a valid employeeId and session token, add the given amount to the user's balance.
